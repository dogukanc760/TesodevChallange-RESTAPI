Api's Mechanics

The images below are the Http requests that can be made for the API and the request schemes accordingly. However, there is one thing we should not forget that all requests in the API can be made by authorized and authenticated users in accordance with the JWT architecture. Therefore, you need to run the database script on GitHub as it is. If the API does not see the necessary authorizations on the user's token on the system, this process results in a 403 Http status code. You should not draw attention to the "nullable" part written in the Schemas section, if the API performs the necessary data validations. In order to be able to operate on the API, you need to log in and get a token and send it with a Bearer - token http request. Otherwise, you cannot do anything on the API.
